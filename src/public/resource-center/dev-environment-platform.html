部署环境

一条流水线，贯穿全部可配置环境

每个环境都有清晰的职责、隔离边界与晋级规则。环境数量与拓扑可按企业规模灵活配置，避免“环境漂移”和“在我机器上能跑”的问题。

⌨

职责清晰

从开发自测到生产运行，每个环境承担明确职责，配置、数据与权限互不串扰。

⇄

逐级晋级

制品只构建一次，按 DEV→SIT→UAT→STAGING→PROD 逐级晋级，杜绝重复打包带来的差异。

⛨

容灾兜底

COB 业务连续性环境提供容灾接管与定期演练，保障关键业务在故障下持续可用。

环境详解

DEV · SIT · UAT · STAGING · PROD · COB

六类环境覆盖研发交付全生命周期，可按需启用、组合与扩展（如新增 PreSIT、PT 性能环境、DR 灾备环境）。

DEV

开发环境

Development

面向开发者的快速迭代环境，允许频繁部署与调试。

  • 特性分支独立部署，秒级反馈
  • 使用统一依赖包与本地脱敏数据子集
  • 仅内部研发可访问，弱审批
频繁变更开发自测低数据敏感
SIT

系统集成测试

System Integration Test

多服务、多团队联调环境，验证系统间接口与集成正确性。

  • 接口契约、服务编排与依赖联通验证
  • 统一脱敏样本数据，保证可重复测试
  • 自动化集成测试随 CI/CD 执行
集成验证接口契约自动化测试
UAT

用户验收测试

User Acceptance Test

业务方与测试方进行验收的类生产环境，贴近真实使用。

  • 业务流程端到端验收
  • 使用规范脱敏的业务样本数据
  • 变更需业务确认，中等审批强度
业务验收类生产受控访问
STAGING

预发布环境

Pre-Production

与生产高度一致的仿真环境，用于上线前最后验证。

  • 生产同构配置、同版本依赖
  • 灰度发布、回归测试与发布演练
  • 严格审批，变更窗口管理
仿真生产灰度回归发布演练
PROD

生产环境

Production

对外提供正式服务的环境，强管控、强审计、强监控。

  • 真实数据，最高数据安全与合规等级
  • 变更需多级审批与发布单
  • 全链路监控、告警与审计留痕
正式服务强管控高敏感数据
COB

业务连续性环境

Continuity of Business

面向容灾与业务连续性的接管环境，保障关键业务不中断。

  • 同城/异地容灾、数据同步与接管
  • 定期切换演练与 RTO/RPO 验证
  • 与 PROD 一致的合规与审计要求
容灾接管切换演练RTO/RPO
研发合规

适配大型企业研发合规与管控分级

不同环境对应不同的访问权限、数据敏感度、审批强度与审计要求，形成清晰可审计的管控矩阵。

环境 数据来源 访问范围 变更审批 审计留痕 合规等级
DEV脱敏子集研发团队弱基础低
SIT统一脱敏样本研发 + 测试弱记录构建与测试低
UAT规范脱敏业务数据业务 + 测试中验收留痕中
STAGING脱敏/受控数据受限发布人员严格发布演练留痕中高
PROD真实生产数据最小授权运维多级全链路审计高
COB生产同步数据最小授权运维多级全链路审计高
⛒

职责分离

开发、测试、发布、运维权限分离，关键环境最小授权，满足审计与内控要求。

▤

变更可追溯

每次晋级与发布都有发布单、审批记录与制品指纹，做到事事有据可查。

✓

合规内建

合规检查、安全扫描与策略门禁内建于流水线，而不是上线前临时补救。

01

基础设施统一依赖包

用统一的基础镜像、依赖清单与制品仓库，消除“环境不一致”这一最常见的交付风险源。

▣

统一基础镜像与运行时

操作系统、运行时、中间件版本由统一基础镜像分发，各环境保持同构。

⛁

统一制品与依赖仓库

私有制品仓库统一托管依赖包，锁定版本与哈希，禁止环境各自拉取不同版本。

⛓

依赖清单与锁文件

以锁文件固化依赖树，构建一次、处处复用，杜绝“同代码不同依赖”。

⛨

供应链安全扫描

对依赖包做漏洞扫描与许可证合规检查，拦截高风险组件进入流水线。

02

脱敏测试样本数据的统一

用统一治理的脱敏样本数据，让测试可重复、可比较，同时守住数据安全与合规底线。

☷

统一脱敏数据集

从生产数据派生出标准化脱敏样本，集中维护，多环境共享同一份基线。

⚿

规则化脱敏

姓名、证件、账号、金额等敏感字段按统一规则脱敏，保持数据可用又不泄密。

↻

可重复与可版本化

样本数据版本化管理，可一键重置环境数据，保障测试结果可复现。

⛒

分级访问控制

不同环境按敏感度获取相应数据，PROD/COB 真实数据严格隔离与最小授权。

统一 CI/CD

一套流水线，统一支撑全环境集成

从代码提交到多环境晋级、发布与回滚，全部由统一 CI/CD 编排，环境差异通过配置而非脚本分叉来管理。

1

代码提交

分支策略 + 代码评审 + 静态扫描门禁。

2

统一构建

使用统一依赖包构建一次，生成带指纹制品。

3

自动化测试

SIT/UAT 注入脱敏样本，跑集成与回归测试。

4

审批晋级

按环境策略审批，制品逐级晋级到 STAGING。

5

发布与回滚

灰度发布到 PROD/COB，异常一键回滚。

⚙

配置即环境

同一流水线通过环境配置区分，避免为每个环境维护一套脚本。

⛿

制品一致性

构建一次、全程复用同一制品，从根本上消除环境间差异。

⤺

可观测与回滚

发布全程可观测，结合健康检查实现快速、安全的自动回滚。
